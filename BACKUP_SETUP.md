# Database Backup Setup

## Обзор

Система бэкапа базы данных состоит из:
- `scripts/backup-database.js` - основной скрипт бэкапа
- `scripts/cron-backup.sh` - обертка для cron с логированием
- Автоматическая очистка старых бэкапов (хранит последние 7)

## Установка и настройка

### 1. Проверка зависимостей

Убедитесь, что установлен `pg_dump`:
```bash
pg_dump --version
```

### 2. Настройка переменных окружения

Проверьте, что в `.env` файле есть `DATABASE_URL`:
```env
DATABASE_URL="prisma+postgres://localhost:51213/?api_key=..."
```

### 3. Тестовый запуск

```bash
# Прямой запуск скрипта
node scripts/backup-database.js

# Или через обертку
./scripts/cron-backup.sh
```

### 4. Настройка Cron задачи

Добавьте в crontab:
```bash
# Открыть редактор crontab
crontab -e

# Добавить строку (запуск каждый день в 3:00)
0 3 * * * /path/to/platform_nest/scripts/cron-backup.sh
```

## Функциональность

### ✅ Реализовано:
- **Автоматическое создание папки backups** - проверяет существование и создает при необходимости
- **Асинхронное выполнение** - использует Promise и setImmediate для неблокирующей работы
- **Парсинг DATABASE_URL** - корректно обрабатывает Prisma формат URL
- **Очистка старых бэкапов** - хранит последние 7 файлов
- **Логирование** - детальный вывод процесса и ошибок
- **Обработка ошибок** - корректная обработка исключений

### 📁 Структура бэкапов:
```
backups/
├── backup-2024-03-18T15-30-00.sql
├── backup-2024-03-17T15-30-00.sql
└── ...
```

### 📝 Логи:
- Консольный вывод с детальной информацией
- Файл `logs/backup.log` для cron результатов

## Команды

```bash
# Ручной бэкап
node scripts/backup-database.js

# Через cron обертку
./scripts/cron-backup.sh

# Проверить cron задачи
crontab -l

# Просмотр логов
tail -f logs/backup.log
```

## Безопасность

- Скрипт использует переменные окружения для подключения к БД
- Старые бэкапы автоматически удаляются для экономии места
- Все операции обернуты в try-catch для предотвращения сбоев
